# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.


## Данные

**IProduct** — модель товара в магазине.

Поля:
* `id: string` — уникальный UUID идентификатор товара;
* `description: string` — описание товара отображаемое в модальном окне;
* `image: string` — URL изображения товара;
* `title: string` — краткое название товара;
* `category: string` — категория товара;
* `price: number | null` — стоимость товара в «синапсах». Может быть `null`, если товар нельзя купить.

**IBuyer** — модель данных покупателя.

Поля:
* `payment: PaymentMethod` — способ оплаты `card` или `cash`;
* `address: string` — адрес;
* `email: string` — электронная почта;
* `phone: string` — телефон.

**IOrder** — модель полного объекта заказа для отправки на сервер.

Состав:
* все поля из `IBuyer`;
* `total: number` — итоговая сумма заказа;
* `items: string[]` — массив идентификаторов выбранных товаров.

**ValidationErr** — объект, содержащий тексты ошибок валидации.

Структура:
* `Partial<Record<keyof IBuyer, string>>` — ключи соответствуют полям формы, значения представляют собой текстовые описания ошибок.

## Модели данных


#### Класс CatalogModel

Назначение: хранение каталога товаров и управление предпросмотром.

Поля:
* `_items: IProduct[]` — массив товаров в каталоге;
* `_preview: IProduct | null` — товар, выбранный для детального просмотра.

Методы и геттеры:
* `setItems(items: IProduct[]): void` — записывает переданный массив товаров в модель и генерирует событие `items:changed`;
* `get items(): IProduct[]` — возвращает полный список товаров в каталоге;
* `getProduct(id: string): IProduct | undefined` — выполняет поиск товара по уникальному идентификатору;
* `set preview(item: IProduct | null)` — устанавливает указанный товар в качестве предпросмотра и генерирует событие `card:select`;
* `get preview(): IProduct | null` — возвращает текущий товар, выбранный для предпросмотра.

#### Класс BasketModel

Назначение: управление состоянием корзины, включая добавление и удаление товаров, а также расчёт стоимости.

Поле:
* `items: IProduct[]` — массив товаров, добавленных в корзину.

Методы:
* `add(item: IProduct): void` — добавляет указанный товар в корзину и уведомляет систему через событие `basket:changed`;
* `remove(id: string): void` — удаляет товар из корзины по его идентификатору;
* `clear(): void` — полностью очищает корзину от всех товаров;
* `getItems(): IProduct[]` — возвращает массив текущих товаров в корзине;
* `getTotal(): number` — вычисляет и возвращает общую сумму стоимости всех товаров в корзине;
* `getCount(): number` — возвращает количество товаров, находящихся в корзине;
* `hasItem(id: string): boolean` — проверяет наличие товара с указанным идентификатором в корзине.

#### Класс OrderModel

Назначение: хранение и проверка данных покупателя, связанных с оформлением заказа.

Поле:
* `order: IBuyer` — объект с данными текущего заказа (способ оплаты, адрес, телефон, почта).

Методы:
* `setField(field: keyof IBuyer, value: string): void` — устанавливает значение указанного поля заказа и генерирует событие `order:changed`;
* `validateOrder(): ValidationErr` — выполняет проверку данных покупателя на заполненность полей. Возвращает объект с текстовыми описаниями ошибок для незаполненных полей. Метод не вносит изменений в состояние модели;
* `getOrderData(): IBuyer` — возвращает текущие данные заказа в виде объекта `IBuyer`;
* `clearOrder(): void` — сбрасывает все данные заказа до начального состояния.

## Слой представления View

Компоненты представления основаны на базовом классе `Component<T>`. Карточки товара используют интерфейс **`ICardActions`** и не имеют зависимостей от моделей или брокера событий. Прочие компоненты (такие как `Header`, `Basket`, `Form`, `Modal`) взаимодействуют с `EventEmitter` и генерируют события в ответ на действия пользователя.

### Базовые компоненты

#### Класс Modal

Реализует универсальное модальное окно. Отвечает за:
* отображение содержимого;
* создание визуального перекрытия страницы;
* блокировку скролла при открытии.

Методы:
* `set content(value: HTMLElement)` — задаёт содержимое окна;
* `open(): void` — запускает отображение модального окна;
* `close(): void` — закрывает модальное окно.

#### Класс Form<T>

Базовый класс для реализации форм. Контролирует валидацию данных и состояние кнопки отправки.

Методы:
* `set valid(value: boolean)` — регулирует доступность кнопки отправки формы;
* `set errors(value: string)` — отображает текст ошибки в специальном поле;
* `render(state: Partial<T> & FormState)` — актуализирует состояние формы, включая ошибки и статус валидности.


### Компоненты страницы

#### Класс Header

Отвечает за визуализацию верхней панели страницы (шапки сайта).

Метод:
* `set counter(value: number)` — обновляет числовое значение счётчика товаров, отображаемого на иконке корзины.

#### Класс Gallery

Обеспечивает визуальное представление сетки товаров на главной странице.

Метод:
* `set items(items: HTMLElement[])` — принимает набор готовых DOM‑элементов карточек и отображает их внутри контейнера галереи.

#### Класс Card

Базовый компонент для карточки товара. В конструкторе задаётся: `constructor(container: HTMLElement, actions?: ICardActions)`.

Обработчик `onClick` привязывается:
* к кнопке `.card__button`, если она присутствует в разметке;
* к корневому контейнеру — в противном случае.

Благодаря этому в режимах превью и каталога клик по свободной области карточки не инициирует действие: реакция возникает только при нажатии на кнопку.

Сеттеры:
* `set title(value: string)` — устанавливает название товара;
* `set price(value: number | null)` — задаёт текстовое представление цены («синапсы» или «Бесценно»);
* `set id(value: string)` — сохраняет ID товара в атрибуте dataset;
* `get id(): string` — возвращает ID товара.

В **CardPreview** переопределён `set price`: при значении `null` кнопка становится неактивной, а на экране появляется надпись «Недоступно».

**Подклассы (наследуют Card):**
* **CardCatalog** — добавляет сеттеры `image` и `category`, включая логику смены цветовых классов;
* **CardPreview** — включает сеттер `description` и управление текстом на основной кнопке (`button`);
* **CardBasket** — реализует сеттер `index` для показа порядкового номера товара в списке корзины.

### Компоненты корзины и заказа

#### Класс Basket

Компонент для визуализации корзины с выбранными товарами.

Методы:
* `set items(items: HTMLElement[])` — обновляет перечень карточек товаров в корзине. При пустой корзине кнопка оформления заказа деактивируется;
* `set total(value: number)` — отображает итоговую сумму заказа.

#### Класс Order (наследует Form)

Форма для выбора способа оплаты и указания адреса доставки.

Методы:
* `set address(value: string)` — задаёт значение адреса;
* `togglePayment(target: HTMLElement)` — переключает активную кнопку способа оплаты.

#### Класс Contacts (наследует Form)

Форма для ввода контактных данных (электронной почты и номера телефона).

Методы:
* `set email(value: string)` — устанавливает значение поля электронной почты;
* `set phone(value: string)` — задаёт номер телефона.

#### Класс Success

Компонент окна подтверждения успешной оплаты.

Методы:
* `set total(value: number)` — показывает итоговую сумму, списанную при оплате;
* `render(data: ISuccess): HTMLElement` — формирует и отображает окно успеха с переданными данными.

## Слой коммуникации

#### Класс ShopApi

Предоставляет методы для взаимодействия с серверной частью интернет‑магазина.

Методы:
* `getLotList(): Promise<IProduct[]>` — запрашивает у сервера актуальный каталог товаров и возвращает промис с массивом продуктов типа `IProduct[]`;
* `orderLots(order: IOrder): Promise<IOrderResult>` — отправляет сформированный заказ на сервер и ожидает ответ с результатом обработки (включая идентификатор заказа и итоговую сумму) в виде промиса типа `IOrderResult`.
