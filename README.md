# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## Данные

**IProduct** — модель товара в магазине.

Поля:
* `id: string` — уникальный UUID идентификатор товара;
* `description: string` — описание товара, отображаемое в модальном окне;
* `image: string` — URL изображения товара;
* `title: string` — краткое название товара;
* `category: string` — категория товара;
* `price: number | null` — стоимость товара в «синапсах». Может быть `null`, если товар нельзя купить.

**IBuyer** — модель данных покупателя. Наследует поля из `IOrderForm` и `IContactForm`.

Поля:
* `payment: PaymentMethod` — способ оплаты (`card` или `cash`);
* `address: string` — адрес;
* `email: string` — электронная почта;
* `phone: string` — телефон.

**IOrder** — модель полного объекта заказа для отправки на сервер.

Состав:
* все поля из `IBuyer`;
* `total: number` — итоговая сумма заказа;
* `items: string[]` — массив идентификаторов выбранных товаров.

**ValidationErr** — объект, содержащий тексты ошибок валидации.

Структура:
* `Partial<Record<keyof IBuyer, string>>` — ключи соответствуют полям формы, значения представляют собой текстовые описания ошибок.

**PaymentMethod** — перечисление допустимых способов оплаты:
* `'card'` — оплата картой;
* `'cash'` — оплата наличными;
* `''` — способ не выбран.

**ApiPostMethods** — допустимые HTTP‑методы для отправки данных:
* `'POST'`;
* `'PUT'`;
* `'DELETE'`.

**EventName** — тип для имени события:
* `string | RegExp` — позволяет подписываться на события напрямую или по шаблону.

**IApi** — контракт API‑взаимодействия.

Методы:
* `get<T>(uri: string): Promise<T>` — выполняет GET‑запрос к указанному URI;
* `post<T>(uri: string, data: object, method?: ApiPostMethods): Promise<T>` — отправляет данные на сервер с указанием HTTP‑метода.

**IApiResponse<T>** — стандартный формат ответа от API.

Поля:
* `total: number` — общее количество элементов;
* `items: T[]` — массив элементов типа `T`.

**IOrderResult** — ответ сервера после успешной отправки заказа.

Поля:
* `id: string` — идентификатор заказа, присвоенный сервером;
* `total: number` — итоговая сумма заказа.

**IProductList** — структура списка товаров, полученного от API.

Поля:
* `total: number` — общее количество товаров в каталоге;
* `items: IProduct[]` — массив товаров.

**IOrderForm** — данные формы оформления заказа.

Поля:
* `payment: PaymentMethod` — выбранный способ оплаты;
* `address: string` — введённый адрес доставки.

**IContactForm** — контактная информация покупателя.

Поля:
* `email: string` — электронная почта покупателя;
* `phone: string` — номер телефона покупателя.

## Модели данных


#### Класс CatalogModel

Назначение: хранение каталога товаров и управление предпросмотром.

Поля:
* `_items: IProduct[]` — массив товаров в каталоге;
* `_preview: IProduct | null` — товар, выбранный для детального просмотра.

Методы и геттеры:
* `setItems(items: IProduct[]): void` — записывает переданный массив товаров в модель и генерирует событие `items:changed`;
* `get items(): IProduct[]` — возвращает полный список товаров в каталоге;
* `getProduct(id: string): IProduct | undefined` — выполняет поиск товара по уникальному идентификатору;
* `set preview(item: IProduct | null)` — устанавливает указанный товар в качестве предпросмотра и генерирует событие `card:select`;
* `get preview(): IProduct | null` — возвращает текущий товар, выбранный для предпросмотра.

#### Класс BasketModel

Назначение: управление состоянием корзины, включая добавление и удаление товаров, а также расчёт стоимости.

Поле:
* `items: IProduct[]` — массив товаров, добавленных в корзину.

Методы:
* `add(item: IProduct): void` — добавляет указанный товар в корзину и уведомляет систему через событие `basket:changed`;
* `remove(id: string): void` — удаляет товар из корзины по его идентификатору;
* `clear(): void` — полностью очищает корзину от всех товаров;
* `getItems(): IProduct[]` — возвращает массив текущих товаров в корзине;
* `getTotal(): number` — вычисляет и возвращает общую сумму стоимости всех товаров в корзине;
* `getCount(): number` — возвращает количество товаров, находящихся в корзине;
* `hasItem(id: string): boolean` — проверяет наличие товара с указанным идентификатором в корзине.

#### Класс OrderModel

Назначение: хранение и проверка данных покупателя, связанных с оформлением заказа.

Поле:
* `order: IBuyer` — объект с данными текущего заказа (способ оплаты, адрес, телефон, почта).

Методы:
* `setField(field: keyof IBuyer, value: string): void` — устанавливает значение указанного поля заказа и генерирует событие `order:changed`;
* `validateOrder(): ValidationErr` — выполняет проверку данных покупателя на заполненность полей. Возвращает объект с текстовыми описаниями ошибок для незаполненных полей. Метод не вносит изменений в состояние модели;
* `getOrderData(): IBuyer` — возвращает текущие данные заказа в виде объекта `IBuyer`;
* `clearOrder(): void` — сбрасывает все данные заказа до начального состояния.

## Слой представления View

Компоненты представления основаны на базовом классе `Component<T>`. Карточки товара используют интерфейс **`ICardActions`** и не имеют зависимостей от моделей или брокера событий. Прочие компоненты (такие как `Header`, `Basket`, `Form`, `Modal`) взаимодействуют с `EventEmitter` и генерируют события в ответ на действия пользователя.

### Базовые компоненты

#### Класс Modal

Реализует универсальное модальное окно. Отвечает за:
* отображение содержимого;
* создание визуального перекрытия страницы;
* блокировку скролла при открытии.

**Конструктор:**
* `constructor(container: HTMLElement)`
* `container: HTMLElement` — контейнер для модального окна.

Поля класса:
* `_container: HTMLElement` (унаследовано от Component) — ссылка на контейнер.
* `_overlay: HTMLElement` — элемент визуального перекрытия страницы.
* `_isOpen: boolean` — флаг состояния окна (открыто/закрыто).

Методы:
* `set content(value: HTMLElement)` — задаёт содержимое окна;
* `open(): void` — запускает отображение модального окна;
* `close(): void` — закрывает модальное окно.

#### Класс Form<T>
Базовый класс для реализации форм. Контролирует валидацию данных и состояние кнопки отправки.

**Конструктор:**
* `constructor(container: HTMLElement, events: EventEmitter)`
* `container: HTMLElement` — контейнер формы;

* `events: EventEmitter` — экземпляр брокера событий.

Поля класса:
* `_submitButton: HTMLButtonElement` — кнопка отправки формы;
* `_errorElement: HTMLElement` — элемент для отображения ошибок.

Методы:
* `set valid(value: boolean)` — регулирует доступность кнопки отправки формы;
* `set errors(value: string)` — отображает текст ошибки в специальном поле;
* `render(state: Partial<T> & FormState)` — актуализирует состояние формы, включая ошибки и статус валидности.

### Взаимодействие слоёв Events

Model и View не связаны напрямую. Модели генерируют события при изменении данных, презентер (main.ts) отслеживает их и обновляет DOM.

**Основные события**
События от (моделей Model → Presenter)

Информируют презентер об обновлениях данных:
* `items:changed` — обновлён каталог товаров;
* `card:preview_changed` — изменён предпросмотр товара;
* `basket:changed` — скорректирован состав корзины;
* `validationErr:change` — обновлено состояние валидации форм.

**UI‑события (генерируются презентером или компонентами с EventEmitter)**

Отправляются в ответ на действия пользователя:
* `card:select` — выбор карточки (payload: { id });
* `card:add` — добавление товара (без payload; товар берётся из catalogModel.preview);
* `card:remove` — удаление из корзины (payload: { id });
* `basket:open` — открытие корзины;
* `order:open` — старт оформления заказа;
* `order:submit` — сохранение данных заказа;
* `contacts:submit` — отправка заказа на сервер;
* `modal:open / modal:close` — управление прокруткой при модальном окне.

## Слой коммуникации

#### Класс ShopApi

Предоставляет методы для взаимодействия с серверной частью интернет‑магазина.

Методы:
* `getLotList(): Promise<IProduct[]>` — запрашивает у сервера актуальный каталог товаров и возвращает промис с массивом продуктов типа `IProduct[]`;
* `orderLots(order: IOrder): Promise<IOrderResult>` — отправляет сформированный заказ на сервер и ожидает ответ с результатом обработки (включая идентификатор заказа и итоговую сумму) в виде промиса типа `IOrderResult`.
